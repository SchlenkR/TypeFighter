Type hierarchies / implicit conversions

Scenarios where one type could be coerced to another — for example where a String is expected but a Number is given. Currently unsupported: unification rejects the mismatch outright. Adding support would require either explicit sub-typing relations or some form of conversion constraint. (TypeFighter does not do anything magical here — there's no automatic ToString or similar. The hypothetical behavior below is purely what the test would expect if conversions were wired in.)
