Expr.Match (Step 6 of TypeSyntaxWithSets)

Patterns supported in this first pass: - Literal (| 0 ->, | "yes" ->, | true ->) - Var (catch-all that binds the scrutinee to a name) - Wildcard (catch-all that ignores the scrutinee) Record / positional patterns will follow once the demand is real. Narrowing happens implicitly: literal patterns emit CHasMember against the scrutinee, so the scrutinee's type closes to the UnionTyp of the literals.

match on numeric literals narrows scrutinee to union

Inferred
Source 
fun x -> match x with | 0 -> "zero" | 1 -> "one"
Inferred 
{0 | 1} -> String
F# test body 
X.Fun (X.Ident "x") (
    X.Match (X.Var "x") [
        X.Arm (X.PatLit 0) (X.Lit "zero")
        X.Arm (X.PatLit 1) (X.Lit "one")
    ])
|> solve [] None
|> shouldSolveType (
    Mono (
        UnionTyp (set [ LiteralTyp (Number 0.0); LiteralTyp (Number 1.0) ])
        ^-> BuiltinTypes.string))

match on booleans covers both arms

Inferred
Source 
fun x -> match x with | true -> 1 | false -> 0
Inferred 
           Bool -> Number
(set { true, false } collapses to Bool per TDef.RecordDefWith / literal-set semantics)
F# test body 
X.Fun (X.Ident "x") (
    X.Match (X.Var "x") [
        X.Arm (X.PatLit true) (X.Lit 1)
        X.Arm (X.PatLit false) (X.Lit 0)
    ])
|> solve [] None
|> shouldSolveType (
    Mono (BuiltinTypes.boolean ^-> BuiltinTypes.number))

match with catch-all Var pattern binds scrutinee

Inferred
Source 
fun x -> match x with | 42 -> "yes" | y -> "other"
Inferred 
           {42} -> String
A Var arm is a catch-all — it binds but adds no constraint on the
scrutinee's type. So the scrutinee only has the literal's member
constraint and narrows accordingly. Explicit widening (via a type
annotation on `x`) would change this.
F# test body 
X.Fun (X.Ident "x") (
    X.Match (X.Var "x") [
        X.Arm (X.PatLit 42) (X.Lit "yes")
        X.Arm (X.PatVar "y") (X.Lit "other")
    ])
|> solve [] None
|> shouldSolveType (
    Mono (
        UnionTyp (set [ LiteralTyp (Number 42.0) ])
        ^-> BuiltinTypes.string))

match with single Var arm is identity

Inferred
Source 
fun x -> match x with | y -> y
Inferred 
           forall a. a -> a
(pure catch-all: the scrutinee is returned as-is)
F# test body 
X.Fun (X.Ident "x") (
    X.Match (X.Var "x") [
        X.Arm (X.PatVar "y") (X.Var "y")
    ])
|> solve [] None
|> shouldSolveType (TDef.Generalize (%1 ^-> %1))

match with wildcard as default arm

Inferred
Source 
match 5 with | 5 -> "five" | _ -> "other"
Inferred 
String
F# test body 
X.Match (X.Lit 5) [
    X.Arm (X.PatLit 5) (X.Lit "five")
    X.Arm X.PatWild (X.Lit "other")
]
|> solve [] None
|> shouldSolveType (Mono BuiltinTypes.string)

match arm body type mismatch fails

Expected error
Source 
match 5 with | 5 -> "five" | 6 -> 42
Error 
arm body types do not unify (String vs Number)
F# test body 
X.Match (X.Lit 5) [
    X.Arm (X.PatLit 5) (X.Lit "five")
    X.Arm (X.PatLit 6) (X.Lit 42)
]
|> solve [] None
|> shouldFail
